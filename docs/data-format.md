# Data Format

## bin/bukhari.json

This is the single source of truth for both the JS and Python packages.

### Top-level shape

```json
{
  "metadata": { ... },
  "chapters":  [ ... ],
  "hadiths":   [ ... ]
}
```

### metadata

```json
{
  "id": 1,
  "length": 7277,
  "arabic": {
    "title":        "صحيح البخاري",
    "author":       "الإمام محمد بن إسماعيل البخاري",
    "introduction": "..."
  },
  "english": {
    "title":        "Sahih al-Bukhari",
    "author":       "Imam Muhammad ibn Ismail al-Bukhari",
    "introduction": "..."
  }
}
```

### chapters[]

```json
{
  "id":      1,
  "arabic":  "كتاب بدء الوحى",
  "english": "Revelation"
}
```

### hadiths[]

```json
{
  "id":        1,
  "chapterId": 1,
  "arabic":    "حَدَّثَنَا الْحُمَيْدِيُّ...",
  "english": {
    "narrator": "Umar bin Al-Khattab",
    "text":     "I heard Allah's Messenger (ﷺ) saying..."
  }
}
```

## chapters/ folder

Generated by `node scripts/build.mjs` from `bin/bukhari.json`.
Used by the JS browser CDN loader and the Python CDN fallback.

```
chapters/
├── meta.json      ← { metadata, chapters, total }
├── 1.json         ← array of hadiths for chapter 1
├── 2.json
└── ...
```

**Do not edit these files manually.** Always edit `bin/bukhari.json`
and regenerate with `node scripts/build.mjs`.
